📝 项目开发文档: (小诸的匠心工坊) - 苹果风格手工博客
1. 项目概述
1.1 目标
搭建一个名为 "小诸的匠心工坊" 的手工博客网站。该网站必须具有高度的美观性，模仿 Apple 官网的视觉逻辑：极简、大气的留白、精致的排版、细腻的过渡动画，以及对产品（手工作品）极致的视觉呈现。

1.2 核心美学原则 (仿苹果风格)
极简主义 (Minimalism): 除非必要，否则不添加任何装饰性元素。让内容（图片和文字）成为主角。

极致留白 (Greate Whitespace): 使用大量的、更有呼吸感的间距，使页面显得高级且不拥挤。

精细排版 (Typography): 标题使用粗体、大号字体，正文使用清晰、易读的字体。层级分明。

微妙动画 (Subtle Animations): 拒绝生硬的显现。所有交互（悬停、加载）都必须是平滑、细腻的淡入淡出或轻微位移。

高质感图片 (High-Quality Imagery): 假定用户上传的都是专业摄影级别的手工图片。

1.3 技术栈
框架: Next.js (App Router, TypeScript)

样式: Tailwind CSS

动画库: Framer Motion (用于实现苹果级别的细腻动画)

部署: GitHub Pages

2. 交互与 UI 设计规范 (Prompt 核心)
此部分旨在作为 Cursor 生成代码时的主要参考指南。

2.1 配色方案 (Color Palette)
背景 (Background): 纯白 (#FFFFFF) 或极淡的灰 (#F5F5F7, 苹果产品背景色)。

文字 (Text): 深灰 (#1D1D1F, 苹果正文字体色) 用于标题，稍淡的灰 (#86868B) 用于副标题或元数据。

强调色 (Accent): 极简的蓝色 (#0071E3) 用于链接或按钮，或使用手工特有的暖木色 (#A27B5C) 仅作点缀。

边框/分割线 (Borders): 极淡的灰 (#D2D2D7)，几乎不可见。

2.2 组件设计规范
容器 (Containers): 保持大宽度（如 max-w-7xl），但在内部保持充足的内边距。

卡片 (Cards):

无描边、无突兀阴影。 默认状态下与背景融为一体，或仅有极细微的背景色差。

悬停效果: 极其轻微的放大（如 scale-101）和微妙的阴影加深，伴随平滑的过渡 (transition-all duration-500 ease-in-out)。

按钮 (Buttons): 胶囊形（Full Rounded）。主按钮为深色或蓝色，次按钮为透明背景带细描边。

导航栏 (Navbar): 半透明玻璃拟态效果 (backdrop-blur-md bg-white/80)，固定在顶部。

3. 分步骤搭建指南 (供 Cursor 执行)
第 1 步: 初始化与核心配置
操作: 打开 Cursor 终端，执行以下命令：

Bash
npx create-next-app@latest artisanhub-blog
(配置选择：Yes TS, Yes ESLint, Yes Tailwind, Yes src/, Yes App Router)

Bash
cd artisanhub-blog
# 安装 Framer Motion 用于动画
npm install framer-motion
第 2 步: 全局样式与 Apple 字体配置
操作: 在 Cursor 中打开 src/app/globals.css。

发送给 Cursor 的 Prompt (Ctrl+K 或对话框):

“请重写这个 globals.css 文件。

设置全局背景色为苹果风格的淡灰 #F5F5F7，文字颜色为 #1D1D1F。

配置 Tailwind，引入 Inter 字体（作为系统字体的美观替代），设置默认字体族。

定义一个 apple-container 的 Tailwind 类，它应该有大宽度（如 max-w-[1440px]）和充足的水平内边距，并且居中。”

第 3 步: 制作半透明导航栏 (Header)
操作: 在 src/components 下新建 Header.tsx。

发送给 Cursor 的 Prompt:

“请创建一个 React 组件 Header.tsx。

它应该是一个固定在顶部的导航栏（sticky top-0）。

使用苹果风格的玻璃拟态效果：半透明背景 (bg-white/80) 和高斯模糊 (backdrop-blur-md)。

左侧是网站 Logo 'ArtisanHub'（使用粗体，稍微宽一点的字间距）。

右侧是简洁的导航链接：首页、作品、关于、联系。

导航链接使用淡灰色，悬停时平滑过渡到深黑色。”

第 4 步: 设计苹果风格首页 (Hero Section & Grid)
操作: 打开 src/app/page.tsx。

发送给 Cursor 的 Prompt:

“请重写这个 page.tsx 主页。

Hero Section: 制作一个巨大的、极简的欢迎区域。标题文字要非常大、非常粗（如 text-7xl font-extrabold），写着：'匠心独运，手作之美。'。下方有一行淡灰色的副标题。

作品网格 (Work Grid): 创建一个三列的网格布局，用于展示手工作品。

Framer Motion 动画: 使用 Framer Motion，让这块网格在页面加载时有一个优雅的、从下往上的淡入效果。

Mock 数据: 使用模拟数据，包含作品图片（找一些极简的手工木工、陶瓷或刺绣图片 URL）、标题、一个极简的标签（如 '木艺'）和简短描述。

调用我之前写好的 WorkCard 组件（如果还没写，请先帮我生成一个超美观的 WorkCard：要求大图片、极简文字、悬停时有微妙的放大和阴影效果）。”

第 5 步: 静态导出配置 (用于 GitHub Pages)
操作: 打开 next.config.mjs。

发送给 Cursor 的 Prompt:

“请修改这个 Next.js 配置文件，以便将其部署到 GitHub Pages。

设置 output: 'export'。

关闭图片优化 images: { unoptimized: true }。

设置 basePath 为我的 GitHub 仓库名，例如 '/artisanhub-blog'（请提醒我替换它）。”

第 6 步: 自动化部署脚本
操作: 新建文件 .github/workflows/deploy.yml。

发送给 Cursor 的 Prompt:

“请帮我写一个完整、标准的 GitHub Actions 脚本（deploy.yml），用于在推送到 main 分支时，自动构建此 Next.js 项目并部署到 GitHub Pages。请确保它使用 node 20 版本，并包含安装依赖、静态构建、以及发布到 gh-pages 分支的步骤。”

第 7 步: Git 推送引导
操作: Cursor 终端。

发送给 Cursor 的 Prompt:

“我已经完成了所有代码。现在请告诉我完整的 Git 命令，帮助我完成以下操作：

初始化 Git。

添加所有文件并提交。

连接到我的远程 GitHub 仓库（提示我输入 URL）。

创建 main 分支并推送到远程。”

第 8 步: 检查与最终优化
操作: 在浏览器中打开部署后的链接。

发送给 Cursor 的对话 Prompt (针对具体页面):

“我觉得现在的作品展示网格间距还是太小了，不够大气。请将 page.tsx 中网格的间距增大，并且让每一个 WorkCard 在悬停时的过渡动画更加缓慢、细腻（比如持续 0.6 秒，使用 ease-out）。”
